Rust Adapter Ingress Map

Where raw provider data enters Acepe and which module owns each edge. Provider quirks stop here — nothing downstream repairs them.

Two ingress modes

ModeWhenInputOutput
Live parse Session connected, streaming ACP session updates (JSON) SessionUpdate → session state engine
History restore Session open / import Disk files (JSONL, SQLite, txt) FullSession → CanonicalTranscriptEvent

Module roles

ModuleRole
acp/providers/<name>/ One home per provider. Documents ingress edge map in mod.rs.
acp/parsers/*_parser.rs Live ACP updates via AgentParser trait.
cursor_history/, session_jsonl/, history/cursor_sqlite_parser.rs Read provider files from disk → FullSession.
session_converter/ FullSession → ProviderOwnedSessionSnapshot + canonical events.
acp/reconciler/ Tool identity authority — shared by live and history paths.
acp/session_materialization/ Join transcript + operations; relink_operations_to_transcript.

Rules

  • Never add provider branches in TypeScript or @acepe/ui.
  • Live and history must agree on tool call ids (normalize_tool_call_id).
  • Classification tables live in acp/reconciler/providers/ — referenced, not duplicated.
  • Fix transcript order in transcript_events.rs or parser — not in UI.
