Canonical Authority Surfaces

Quick lookup: who owns what in Acepe's session model. Product truth lives in Rust; TypeScript and @acepe/ui consume it.

Surface Owns Rust home
SessionStateGraph lifecycle, activity, turn state, capabilities, revision acp/session_state_engine/graph.rs
Canonical transcript message order, display ids, text/thought/tool sequence acp/transcript_projection/
Operation graph tool call state, args, results, parent/child links acp/projections/ + session_materialization
Interaction graph questions, approvals, blocking user interactions acp/projections/
Tool identity kind, canonical name, argument interpretation acp/reconciler/
Provider adapters raw provider → canonical facts (only place for quirks) acp/providers/<name>/, acp/parsers/
Session open full canonical state at open + last_event_seq boundary acp/session_open_snapshot/
Scene model (TS) domain → view props mapping (no truth) agent-panel-graph-materializer.ts
View presentational render only packages/ui/src/components/agent-panel/

provider_row_idProvider row uuid — metadata

Forbidden patterns

  • canonical ?? hotState fallbacks in TypeScript
  • Sorting or reordering transcript rows in Svelte/components
  • Provider-specific branches in UI code
  • Using raw provider message ids as display entry ids
  • Tauri or store imports inside @acepe/ui
